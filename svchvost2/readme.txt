АЛГОРИТМ РАБОТЫ ВО ВРЕМЯ ОБНОВЛЕНИЯ СЕРВЕРА ДО НОВОЙ ВЕРСИИ.

1) Старый сервер скачивает файл с новой версией сервера к себе
   и помещает файл в свой рабочий каталог  (asd.dat).

2) Старый сервер выгружает из своего тела программу su.exe и
   тоже кладет ее с свой рабочий каталаг, затем ее запускает
   с параметрами: update svchvost.exe svchvost.exe asd.dat

3) Программа su.exe, в свою очередь, заменяет содержимое файла
   svchvost.exe содержимым файла asd.dat (новая версия)

4) Если новая версия сервера является шаттлом (контейнером),
   то она сначала удаляет su.exe, затем записывает туда
   новую ее версию. Затем она использует программу su.exe для
   запуска настоящего сервера, для чего сначала выгружает из
   своего тела необходимые DLL, затем опять файл asd.dat.

5) Происходит финальный запуск программы su.exe, которая
   запускает сервер
